Detailed Description

Class for a FILE block. Used mainly in agent files, a FILE block can contain any type of data These blocks are primarily used for sprite, body data and genetics files. You can only tell which type of file it is by looking at the file's name.

PrayBlock constructor.

Constructs a new PrayBlock, setting the PrayBlock's name, content, flags, and type.

Parameters:
$prayfile If constructing by reading a PRAY file, is a PRAYFile object. Otherwise is allowed to be anything (it's assumed the subclasses take care of it)
$name The name of the block. Must be unique within the PRAYFile. This will be checked by the PRAYFile.
$content If constructing the PrayBlock by reading a PRAY file, must be a binary string. Otherwise, should be null (but doesn't have to be).
$flags A 1-byte integer containing the flags this PrayBlock has set
$type The type of the PrayBlock as a PRAY_BLOCK_* constant. Must be a four-character string.
